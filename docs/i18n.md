# ローカライズ (i18n) 実装ガイド

## 概要

本ゲームは日本語（ja）と英語（en）の2言語に対応している。
言語はタイトル画面のEN/JAボタンで切り替え可能で、選択は `localStorage` に永続化される。

## アーキテクチャ

### コアモジュール: `src/i18n/index.ts`

| 関数 | 用途 |
|------|------|
| `getLang()` | 現在の言語コード (`'ja'` \| `'en'`) を返す |
| `setLang(lang)` | 言語を切り替え、localStorage に保存し、リスナーを通知 |
| `onLangChange(fn)` | 言語変更時のコールバックを登録（解除関数を返す） |
| `t(key, params?)` | UIテキストを取得。`{{param}}` 形式の補間に対応 |
| `label(category, key)` | データラベルを取得（派閥名・場所名・性格名など） |

### ファイル構成

```
src/i18n/
  index.ts            # コアモジュール（getLang/setLang/t/label）
  ja.ts               # 日本語UIテキスト辞書
  en.ts               # 英語UIテキスト辞書
  ja-data.ts          # 日本語データラベル（派閥・場所・性格 等）
  en-data.ts          # 英語データラベル
  en-students.ts      # 英語生徒名・ニックネーム・説明文
  en-dialogue.ts      # 英語会話セリフ（説得バトル・雑談 等）
  en-catchphrase.ts   # 英語キャッチフレーズ
```

## テキストの種類と実装パターン

### 1. UIテキスト — `t(key, params?)`

画面上のラベル・ボタン・メッセージなど。テンプレート内で直接呼び出す。

```typescript
import { t } from '../i18n';

// 基本
t('title.gameStart')  // → "ゲームスタート" / "Start Game"

// パラメータ補間
t('game.deliverLostResult', { name: '田中', item: 'ノート' })
// → "田中にノートを届けた。" / "Delivered Notebook to Tanaka."
```

**辞書ファイル**: `ja.ts` / `en.ts`
キーが英語辞書に存在しない場合、日本語辞書にフォールバックする。

### 2. データラベル — `label(category, key)`

派閥名・場所名・性格名・部活名など、マスターデータに紐づく表示名。

```typescript
import { label } from '../i18n';

label('faction', 'conservative')  // → "保守派" / "Traditionalists"
label('personality', 'passionate') // → "情熱的" / "Passionate"
```

**辞書ファイル**: `ja-data.ts` / `en-data.ts`
カテゴリは `DataCategory` 型で定義（`hairstyle`, `hobby`, `faction`, `location` 等）。

### 3. 生徒名 — `getStudentName()` / `getStudentNickname()`

`src/data/students.ts` で定義。`getLang()` を参照して英語名を返す。

```typescript
import { getStudentName, getStudentNickname } from './data/students';

getStudentName(student)     // → "田中 太郎" / "Taro Tanaka"
getStudentNickname(student) // → "太郎" / "Taro"
```

**英語データ**: `en-students.ts` の `EN_STUDENT_DATA`

### 4. 性格×性別セリフ — `eventLines.ts`

落とし物・おつかいイベントのセリフ。日本語は性格×性別で口調が変わる（5性格×2性別＝10バリエーション）。
英語モードでは `t()` の汎用セリフにフォールバックする。

```typescript
import { getLostItemThanks } from './data/eventLines';

// 日本語: 性格×性別に応じたセリフをランダム選択
// 英語: t('game.deliverLostOwnerLine') を返す
getLostItemThanks(personality, gender)
```

同様のパターン: `getErrandThanks`, `getErrandRelayThanks`, `getErrandRequest`,
`getErrandFollowUp`, `getPlayerDeliverLost`, `getPlayerDeliverErrand`, `getPlayerAcceptErrand`

### 5. 会話・バトルセリフ — 各データモジュール

- `src/data/conversationLines.ts`: 会話セリフ（`getLang()` で英語データに切り替え）
- `src/data/battleLines.ts`: 説得バトルセリフ（同上）
- `src/catchphrase.ts`: キャッチフレーズ（同上）

## 新しいテキストを追加する手順

1. **日本語テキスト**を該当箇所に実装する
2. **`ja.ts`** にキーと日本語テキストを追加（UIテキストの場合）
3. **`en.ts`** に同じキーで英語テキストを追加
4. コード内で `t('キー名')` を使用する
5. データラベルの場合は `ja-data.ts` / `en-data.ts` に追加し、`label()` を使用

### キー命名規則

```
{画面名}.{要素名}          例: title.gameStart, battle.roundLabel
{画面名}.{動作}{対象}      例: game.deliverLostResult, game.errandLog
```

## 言語切り替えと画面再描画

`setLang()` 呼び出し時に `onLangChange` リスナーが通知される。
現状、タイトル画面は手動で `render()` を呼び出して再描画する。
ゲーム中の画面遷移時は新しい画面が `t()` の最新言語で描画されるため、
基本的にはゲーム開始前にタイトル画面で言語を選択する運用を想定している。

## フォールバック規則

- `t(key)`: `enStrings[key]` → `jaStrings[key]` → `key`（キー文字列そのまま）
- `label(cat, key)`: `enData[cat][key]` → `jaData[cat][key]` → `key`
- 生徒名: `EN_STUDENT_DATA[id]?.name` → `student.name`（日本語名）
- eventLines: 英語モード時は `t()` にフォールバック、日本語は性格×性別セリフ
