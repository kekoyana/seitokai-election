docs/worldview.md を読み込み、以下の観点でコードベースを検査してください。
検査結果を一覧で報告し、違反があれば具体的なファイル名・行番号・修正案を提示してください。

## 検査項目

### 1. ログテキストの世界観チェック（重要度: 高）

`src/game.ts` および `src/battleLogic.ts` 内のログ生成テキストを確認:

- **禁止ゲーム用語**: 「ダメージ」「HP」「バフ」「デバフ」「パス」「ターン」「ラウンド」がログ文字列中に含まれていないか
- **システム的すぎる表現**: 「○○が判明！」「○○を獲得！」のような通知的な書き方がないか
- **数値のみの表示**: 描写文なしに「好感度 +5」のような数値だけのログがないか
- **一人称代名詞**: ログ文中に「僕」「私」「俺」が使われていないか（主語省略が原則）

### 2. 効果色の整合性チェック（重要度: 高）

コード全体で以下の色が正しく使われているか:

- ポジティブ効果: `#7EC850`（緑）
- ネガティブ効果: `#F07070`（赤）
- 中立・軽微: `#999`

`src/` 内で `style="color:` を含むログ生成箇所を検索し、上記以外の色が効果表示に使われていないか確認。
※ UI要素の装飾色（派閥色、好感度表示色など）は検査対象外。ログ内の `<span style="color:...">` のみ対象。

### 3. 派閥色の一貫性チェック（重要度: 中）

`src/data.ts` の `CANDIDATE_INFO` で定義された各派閥のイメージカラーを確認し、
`src/screens/` 内で「味方」「支持」を表す色として別の色（特に `#27AE60` 緑）が使われていないか検査。

味方/支持の表示には固定色ではなく、常にその派閥のイメージカラー（`CANDIDATES` の `color` プロパティ）を使うべき。

### 4. セリフのトーンチェック（重要度: 低）

`src/game.ts` や `src/battleLogic.ts` 内にセリフ（「」で囲まれた文字列）がある場合、
worldview.md の性格別トーン表と照合して明らかに不自然なものがないか軽く確認。
（完全な自然言語チェックは不要。明らかな違和感のみ報告）

```
- 派閥色の一貫性: OK
```

違反が0件の場合は「全項目問題なし」と報告してください。
