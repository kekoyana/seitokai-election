# 生徒会長投票ゲーム (seitokai-election)

学園生活シミュレーション × 説得バトルゲーム。
プレイヤーは学園の一生徒として、3つの派閥のいずれかを支持し、30日後の学園祭企画投票での勝利を目指します。

## 🎮 ゲーム概要

今まで内部向けだった学園祭を「外部公開」するかどうか。学園を二分する議論が巻き起こります。
プレイヤーは「保守」「革新」「体育」のいずれかの派閥に属し、他の生徒たちと交流・説得を行うことで、学園全体の支持を取り付けていきます。

### 3つの派閥
- **保守派**: 伝統的な内部イベントを守るべき（中心人物: 鷹山 誠一）
- **革新派**: 外部公開して学園を活性化すべき（中心人物: 結城 あかり）
- **体育派**: 学園祭を体育祭にし、他校と競い合うべき（中心人物: 獅堂 剛）

### 主なシステム
- **組織票システム**: 個人票ではなく、クラスや部活といった「組織」単位の支持が勝敗を決します。組織の性格（独裁型/衆議型など）を見極め、キーパーソンを説得することが重要です。
- **説得バトル**: 「態度」「話題」「立場」を組み合わせた3段階のコマンドで相手の思想を動かす、バーグラフ綱引き形式のバトル。
- **活動家システム**: プレイヤー以外のNPCも「活動家」として日々他の生徒を説得して回ります。情勢は常に動的に変化します。

## 🛠 技術スタック

- **言語**: TypeScript (Strict mode, `any`禁止)
- **レンダリング**: Vite + Vanilla CSS (DOM操作ベース。Phaser等のゲームエンジンは不使用)
- **対応デバイス**: PC + スマートフォン（Pointer Eventsによる操作）
- **画像生成**: ローカルComfyUI (Illustrious-XL) によるキャラクターポートレート生成

## 🚀 開発コマンド

```bash
# 依存関係のインストール
npm install

# 開発サーバーの起動 (http://localhost:5173)
npx vite

# 型チェック
npx tsc --noEmit

# プロダクションビルド
npx vite build

# 説得バトルのロジックシミュレーター実行
npx tsx tools/simulate.ts
```

## 📝 開発上の注意

本プロジェクトでは、ルートディレクトリの AI 向けガイドと `docs/` 以下の仕様書を「Single Source of Truth」として扱います。

- **用語の統一**: `docs/glossary.md` に従い、「人物」ではなく「生徒」、派閥名は「保守/革新/体育」と呼称してください。
- **世界観**: ログやセリフの文面は `docs/worldview.md` のトーン＆マナーを遵守してください。
- **UIデザイン**: `docs/design-guide.md` のCSS変数やコンポーネント設計に従ってください。
- **ドキュメント更新**: 仕様変更や構成変更を行った際は、必ず関連する `docs/` と AI 向けガイドを更新し、`docs/README.md` から辿れる状態を維持してください。
